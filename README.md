# Descripción del Sistema Web Propuesto

## Introducción

El **Sistema de Gestión de Recursos Operativos, Administrativos y de Seguridad para Cooperativas de Transporte Nacional (SGROAS)** es una aplicación web desarrollada para optimizar la administración de los recursos operativos, administrativos y de seguridad de las cooperativas de transporte interprovincial del Ecuador.

Este proyecto responde a cuatro preguntas fundamentales:

* ¿Qué problema resuelve?
* ¿Quiénes son los usuarios?
* ¿Por qué se implementa como una aplicación web?
* ¿Cuál es el alcance del sistema?

---

## Problema que Resuelve

Las cooperativas de transporte interprovincial en Ecuador constituyen un servicio esencial para la movilidad de millones de personas entre ciudades y comunidades rurales. Sin embargo, según un informe de la **Agencia Nacional de Tránsito del Ecuador (2022)**, más del **60 %** de estas cooperativas gestionan sus operaciones mediante registros físicos o herramientas ofimáticas no integradas, lo que genera:

* Errores administrativos.
* Pérdida de información.
* Dificultades en la supervisión operativa.

Además, el personal administrativo dedica, en promedio, más del **40 %** de su tiempo a tareas de registro manual que podrían automatizarse (García et al., 2018).

Para solucionar esta problemática, el sistema centraliza la gestión de:

* Vehículos.
* Conductores.
* Rutas.
* Asignaciones.
* Incidentes operativos.
* Reportes administrativos.

Todo ello mediante una única plataforma digital.

---

## Usuarios del Sistema

El sistema está diseñado para tres tipos de usuarios:

### Administrador del Sistema

Responsable de:

* Gestionar usuarios.
* Administrar los accesos.
* Supervisar el registro de auditoría.

### Coordinador de Ruta

Encargado de:

* Registrar unidades de transporte.
* Registrar conductores.
* Administrar rutas.
* Gestionar las asignaciones de unidades y conductores.

### Personal de Seguridad

Responsable de:

* Registrar incidentes operativos.
* Consultar reportes del sistema.

---

## ¿Por qué una Aplicación Web?

Se implementa como una aplicación web porque ofrece múltiples ventajas:

* Acceso desde cualquier dispositivo con navegador web.
* No requiere instalación.
* Permite el trabajo simultáneo de varios usuarios.
* Facilita el acceso desde distintas ubicaciones.
* Reduce los costos de despliegue y mantenimiento frente a una aplicación de escritorio.

---

## Alcance del Proyecto

Durante este semestre se desarrollarán los siguientes módulos:

1. **Autenticación y gestión de usuarios.**
2. **Gestión operativa**, que incluye:

   * Unidades.
   * Conductores.
   * Rutas.
3. **Planificación**, mediante la asignación de unidades y conductores a las rutas.
4. **Registro de incidentes operativos.**
5. **Inteligencia y análisis**, incorporando clasificación automática de la gravedad de los incidentes mediante Inteligencia Artificial.
6. **Reportes**, con exportación en formatos **PDF** y **Excel**.

Estos módulos cubren los requisitos funcionales de mayor prioridad identificados durante el proceso de Ingeniería de Requisitos realizado en el semestre anterior y conforman un núcleo funcional coherente, desplegable y de alto valor para una cooperativa de transporte nacional.
